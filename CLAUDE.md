# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- `pnpm dev` — start dev server on port 3000
- `pnpm build` — production build (Vite + Nitro, output in `.output/`)
- `pnpm preview` — preview the production build
- `pnpm test` — run Vitest once (`pnpm test -- <pattern>` to filter, `pnpm exec vitest` for watch mode)
- `pnpm check` — Biome lint + format check (also `pnpm lint`, `pnpm format`)

Production server: `node .output/server/index.mjs` after `pnpm build` (respects `PORT`, defaults to 3000). Nitro emits the deployable artifact to `.output/` using the default node-server preset; set `NITRO_PRESET` for a specific host (e.g. `vercel`, `cloudflare`).

## Architecture

TanStack Start app (React 19 + Vite 8) using file-based routing, server-rendered through Nitro.

- **Entry:** `src/router.tsx` constructs the TanStack Router with `defaultPreload: 'intent'`. The `Register` module augmentation makes the router type globally inferable.
- **Routes:** `src/routes/` is the file-based route tree. `__root.tsx` defines the HTML shell (`shellComponent`), head metadata, and mounts TanStack Devtools. New routes are added as files here; `src/routeTree.gen.ts` is auto-generated by the `@tanstack/router-plugin` Vite plugin — never edit it (also Biome-ignored).
- **Server functions / API routes:** Use `createServerFn` from `@tanstack/react-start` and the `server.handlers` property on file routes (see README for examples).
- **Build pipeline:** `vite.config.ts` chains `devtools()`, `nitro()`, `tailwindcss()`, `tanstackStart()`, `viteReact()`. Nitro is the SSR adapter; Sentry packages are externalized.
- **Styling:** Tailwind CSS v4 via `@tailwindcss/vite`; global stylesheet at `src/styles.css` is loaded by the root route via `?url` import.
- **Path aliases:** `#/*` and `@/*` both resolve to `src/*` (tsconfig + package.json `imports`).

## Conventions

- Biome 2.x for lint/format: **tabs** for indent, **double quotes** in JS/TS, import organization on. `src/routeTree.gen.ts` and `src/styles.css` are excluded.
- TypeScript is strict with `noUnusedLocals`, `noUnusedParameters`, `verbatimModuleSyntax`, and `noUncheckedSideEffectImports` — use `import type` for type-only imports.

## Design & UI

- **Always invoke `/design-taste-frontend`** before building or refining any UI component or page. This is non-negotiable — it keeps the output out of AI slop territory.
- **No monochrome color schemes.** The palette is multi-hue: coral primary + honey yellow secondary + warm off-white surfaces. Never default to a flat grayscale-only scheme.
- **Font stack:** `Geist Variable` for all UI and body text, `Fraunces Variable` for display/hero headlines. Both are already installed via `@fontsource-variable`. Never use Inter or any system default sans.
- **Color system** lives in `src/styles.css` as CSS custom properties, surfaced to Tailwind v4 via `@theme inline`. Key tokens: `--primary` (coral `oklch(0.598 0.155 28.8)`), `--accent` (yellow `oklch(0.855 0.12 88)`). Always reference these tokens rather than hardcoded colors.
- **Component library: shadcn/ui (Base UI, new-york style).** `components.json` is configured. Add new components via `pnpm dlx shadcn@latest add <name>`. Never write a custom headless component if shadcn already provides one — use what exists for consistency.
- **Motion is expected.** Invoke `/web-animation-design` for any animated interaction. Use `tw-animate-css` (already imported in `styles.css`) for Tailwind-native animations. Prefer subtle, purposeful motion over static UI.
- **Images: product closeups only.** This is an ecommerce — images should show the product clearly. Source from Unsplash (`images.unsplash.com`) or Pinterest. No decorative lifestyle filler. Pick images contextually relevant to the specific product or category shown.

## Copywriting

- **Never use " — " (em dash) or " - " (hyphen) as a sentence connector in UI copy.** It is the single most recognizable AI slop tell. Rewrite the sentence instead.
- **No bullet-point-heavy body copy.** If the content can be a real sentence, write a real sentence.
- **Brand voice: warm and reachable.** Write for every visitor regardless of vet knowledge — plain language, no clinical jargon unless the context demands it, short sentences, conversational but not childish.
- **Target customer: retail consumers** buying pet health or veterinary products. Their primary emotion is care and concern for their animal. Copy should speak to that, not to a clinical professional audience.
- **Use `/marketing-skills:marketing-ideas`** when brainstorming product positioning, feature naming, campaign angles, or any copy that needs to persuade, not just describe.
